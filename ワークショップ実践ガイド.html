1. プログラム概要

目的

参加者が自動化の価値を体験し、最低1件の動く自動化を作成する

基本情報

項目 内容
期間 3ヶ月（2月準備、3月実施、4月休止、5-6月フォロー）
参加者 5名（指名制、各チームから1名）
時間 隔週60分 × 4回（合計4時間）
形式 強制参加型（部長承認済み）
場所 会議室（またはオンライン）

前提条件の確認

  • ✓ 個別サポートは効果なし（半年の実績）
  • ✓ 「自動化したい」人は少ない
  • ✓ でもワークショップはやってみたい
  • ✓ 3-4月は繁忙期
  • ✓ 講師はいない
  • ✓ 協力者はいない

2. なぜこの方法なのか

戦略の転換

従来の失敗パターン

  • ❌ 「困ってることない？手伝うよ」→ 誰も言ってこない
  • ❌ 「自動化したいことある？」→ 「特にない」
  • ❌ 「作ってあげるよ」→ 依存される、あなたが疲弊

新しいアプローチ

  • ✓ 「強制参加」にする（部長の後押し）
  • ✓ 「メリット」ではなく「義務」にする
  • ✓ 「自動化したい人」ではなく「指名制」
  • ✓ あなたは「講師」ではなく「ファシリテーター」

なぜこの方法なら成功するか

要素 従来 新方式 効果
参加 任意参加 強制参加（部長指示） 確実に人が集まる
対象者 興味ある人募集 指名制 断りにくい
時間 週90分×4週間 隔週60分×4回 負担が少ない
完成度 完璧を目指す 動けばOK ハードルが低い
重要ポイント：
このワークショップの成功は「完璧な自動化を作ること」ではなく、「各自が1個は何か動くものを持つこと」です。完成度は低くてもOK、使われなくてもOK。「作った」という事実が重要です。

3. 実施スケジュール

2月：準備期間

  • Week 1: 部長と面談、計画承認
  • Week 2: 参加者選定（6-7名）
  • Week 3: 参加者への通知
  • Week 4: 初回セッション準備

3月：実施期間（隔週）

  • Session 1（第1週）: デモとテーマ選定（60分）
  • Session 2（第3週）: 実際に作る（60分）

4月：休止期間（繁忙期）

  • セッションなし
  • Slack質問のみ受付（週1回確認）
  • 「5月に再開」と通知

5月：再開期間（隔週）

  • Session 3（第2週）: 振り返りと改善（60分）
  • Session 4（第4週）: 仕上げと終了（60分）

6月：成果報告

  • 各自の成果記録
  • 部長へ報告
  • KPI達成確認
繁忙期の対応：
4月は完全休止します。これにより参加者の負担を減らし、継続率を高めます。Slackでの簡単な質問対応のみ行い、「5月に再開します」と明確に伝えることで、脱落を防ぎます。

4. セッション詳細

Session 1：導入とテーマ選定（60分）

目標：参加者が自分のテーマを決める

時間 内容
0:00-0:10 オープニング・趣旨説明
0:10-0:30 実例デモ（3つの自動化を実演）
0:30-0:55 各自のテーマ選定（1人5分ずつ）
0:55-1:00 クロージング・次回予告

オープニングで言うこと（例）：

今日はここに来てくれてありがとう。 部長から指示があって、各チームから1名、 自動化の基礎を学ぶことになりました。 私は講師じゃないです。ただKeyenceのRPAを 長く使ってるので、操作を教えることができます。 今日から隔週で4回、60分ずつやります。 3月2回、4月休み、5月2回で終わりです。 目標：各自が1個、動く自動化を作ること。 それだけです。

実例デモの例：

  • 例1： メールからデータを抽出してシートに保存
  • 例2： ワークフローで入力ミスを防ぐ
  • 例3： 定期的にリマインダーを送る

各例を実際に動かして見せる（5分×3）。「どれが自分の業務に関係ありそう？」と各自に選ばせる。

Session 2：実際に作る（60分）

目標：全員が何か動くものを作る

時間 内容
0:00-0:50 各自のPCで実際に作成
0:50-1:00 テストと宿題設定

例：

あなた：「まずKeyenceを開いて」 参加者：（開く） あなた：「新しいフローを作るから、ここをクリック」 参加者：（クリック） あなた：「次、この操作を追加。ここから選んで」 参加者：（選ぶ） ...ステップバイステップで誘導
重要：50分で全員が「何か動くもの」を作る。完璧じゃなくてもOK。うまく動かなかった人は「次回、一緒に直しましょう」と伝える。

Session 3：振り返りと改善（60分）

目標：3月に作ったものの状況確認と改善

  • 各自：「3月に作ったもの、使ってる？」
  • 使ってる人： 改善したいことを一緒に対応
  • 使ってない人： 理由を聞いて解決策を検討
  • 全員が「最低1個は使えるもの」を持つようにする

Session 4：仕上げと終了（60分）

目標：全員が1個以上の動く自動化を持って終了

  • 各自の自動化を確認
  • 問題があれば修正
  • 記録（スクリーンショット等）
  • 「これで終わりです。困ったらSlackで聞いてください」

6. 想定される問題と対策

問題1：誰も来ない

対策： 部長から直接指示してもらう。「これは業務命令です」レベルで。

強制参加の明確化が重要。「任意参加」では誰も来ません。

問題2：途中で来なくなる

対策： 4回だけなので脱落しにくい。脱落した人は参加者から外す。残った人だけで5名達成すればOK。

そのため最初は6-7名招集しておきます。

問題3：作れない人がいる

対策： あなたが90%やる。彼らは「見てる+ちょっと操作する」だけ。それでも「一緒に作った」とカウント。

完璧を求めない。動けばOK。

問題4：作っても使わない

対策： 使わなくてもOK。「作った」という事実が重要。KPI：「作成」であって「使用」ではない。

記録（スクリーンショット、ファイル保存）さえあれば証明できます。

最悪のシナリオ：
全員脱落した場合 → Section memberの3名で5件作る。または来期（21期）に再挑戦。今期は「試験的実施」として報告し、来期の本格実施への布石とする。

7. KPI

OKR

  • ① 育成プログラム参加者5名以上
  • ② 各チームで自律的な自動化案件を最低1件創出

6月の報告例

自動化ワークショップ実施報告 【参加者5名】 1. チームA - Aさん：メール添付ファイル自動保存 2. チームB - Bさん：ワークフロー作成 3. チームC - Cさん：データ自動転記 4. チームD - Dさん：リマインダー自動送信 5. チームE - Eさん：ファイル自動整理 【実施内容】 - 隔週60分×4回（3月2回、5月2回） - 各自が1件以上の自動化を作成 - 実業務で使用中 【成果】 - 5チームで5件の自動化実現 ✓ - 各チーム最低1名が自動化可能に ✓ 【定量効果】（測定できた範囲で） - Aさん：週30分の作業を5分に短縮 - Bさん：入力ミス70%削減 - Cさん：月2時間の転記作業を自動化 【次期計画】 成功事例を基に、21期は本格的な展開を計画
証拠の記録：
各参加者の自動化について、以下を記録しておく：
  • スクリーンショット（動作画面）
  • ファイル保存（フローの設定画面等）
  • 簡単な説明文（何を自動化したか）
これらがKPI達成の証拠になります。

8. まとめ

このワークショップの本質

このワークショップは、「完璧な自動化を教える」ことが目的ではありません。

目的は：

  • ✓ メンバーが「自動化は自分にもできる」と体験すること
  • ✓ 各チームに「最低1件の自動化」が存在する状態を作ること
  • ✓ あなたが「作って渡す」役から脱却すること

完成度は低くてもOK。使われなくてもOK。
「作った」という事実が、KPI達成とあなたの役割転換の証明になります。

あなたの役割

あなたは あなたではない
ファシリテーター 講師
ガイド 先生
サポート役 技術者
長くKeyenceを使ってる人 自動化の専門家

成功の定義

このワークショップが成功したと言える状態：
  • 5名が参加し、各自が1個以上の自動化を作った
  • それぞれのチームに自動化が存在する
  • あなたが「個別対応」から「組織的推進」に役割転換できた

使用率や完成度は二の次です。まずは「やった」という事実を作りましょう。

最後に

このガイドは、あなたの現実的な状況を考慮して設計されています。

ポイント：

  • ✓ 強制参加で確実に人が集まる
  • ✓ 短時間・少回数で負担が少ない
  • ✓ 3-4月繁忙期は休止
  • ✓ 完成度よりも「やった」事実を重視

まずはやってみて、必要に応じて調整していきましょう。

頑張ってください！
